# Research Data Repository — DSpace PoC

Self-hosted research data repository on a single VM with Docker Compose, using DSpace and the Iris dataset as a demo.

## Architecture
```
                   ┌──────────┐
Browser / API ───► │  Nginx   │ :80
                   └────┬─────┘
           ┌────────────┴────────────┐
           ▼                         ▼
      ┌─────────┐             ┌──────────┐
      │ web-ui  │:8080        │ web-api  │:8000
      │ (Spring Boot) │       │ (Spring Boot) │
      └────┬────┘             └────┬─────┘
           └──────────┬───────────┘
                       ▼
      ┌────────────────────────────────────────┐
      │  PostgreSQL  Solr  Redis  RabbitMQ     │
      └────────────────────────────────────────┘
                       ▼
                   ┌────────┐
                   │  MinIO │  (S3-compatible object storage)
                   └────────┘
```

## Services
| Service | Image | Role |
|---------|-------|------|
| **nginx** | nginx:1.25-alpine | Reverse proxy, static files |
| **web-ui** | dspace/dspace (local build) | User-facing UI |
| **web-api** | dspace/dspace (local build) | REST API |
| **worker** | dspace/dspace (local build) | Background jobs placeholder |
| **db** | postgres:15-alpine | Metadata storage |
| **search** | dspace/dspace-solr | Solr search cores |
| **cache** | redis:7-alpine | Cache/session |
| **mq** | rabbitmq:3.12-management-alpine | Message broker |
| **minio** | minio | S3-compatible asset store |

## Prerequisites
- Docker ≥ 24 and Docker Compose v2
- 4 CPU cores, 8 GB RAM, 50 GB free disk
- Ports: 80, 8000, 8080, 9001 (MinIO console), 15672 (RabbitMQ)

## Docker setup & access
```bash
git clone <this-repo>
cd DSpace
cp .env.example .env    # set DSPACE_DB_PASSWORD and admin credentials if desired
make build              # build the DSpace image locally
make setup              # start infra, migrate DB, create admin, seed demo community/collection, reindex, start services

# Check containers and health
make status
```
Access:
- Angular UI: http://localhost:8080/ (via Nginx)
- REST HAL browser: http://localhost:8080/server/ (proxied to API; login: admin@example.com / admin)
- Reverse proxy: http://localhost/ (routes / to UI, /server/ to API)
- Direct API port: http://localhost:8000/server
- Demo collection handle created by setup: `123456789/2` (used by ingest)

## Ingest the sample Iris dataset
```bash
# Optional token for auditing output; ingest uses admin credentials inside the container
make ingest token=demo
```
What it does:
- Builds a Simple Archive Format (SAF) package from `datasets/iris`
- Imports into collection `123456789/2`
- Refreshes the discovery index

## Useful make targets
```bash
make up         # start all services
make down       # stop services
make logs       # tail all logs
make logs-web-ui# tail UI logs
make shell      # shell inside web-ui
make cli cmd="database info"  # run any DSpace CLI command
make clean      # stop and remove volumes (factory reset)
```

```
├── metadata.json     # Record metadata (used by ingest script)

```
